03 / Engineering decisions

One product language, many applications

Repeated interface behavior slowed every new product. So: typed Vue primitives, tokens, and composables built once. Each product keeps its own look. The shared layer stays small — product workflows stay with the product.

Vue · Nuxt · TypeScript · CSS tokens

Small services, legible boundaries

Integrations rot when one service accumulates implicit jobs. So: focused Go gateways, PostgreSQL, explicit contracts, predictable failures. More interfaces to maintain. Fewer surprises.

Go · PostgreSQL · HTTP contracts

Make delivery reproducible

Infrastructure knowledge dies in shell history. So: versioned OpenTofu, Ansible, containers, secrets tooling, CI. Automation costs more than a one-off command. It pays back on every rebuild.

Linux · OpenTofu · Ansible · Docker · CI/CD
